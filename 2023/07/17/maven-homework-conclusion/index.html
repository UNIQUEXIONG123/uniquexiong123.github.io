本博客用于总结开发maven插件过程的一些过程

项目名称

boot-jar-plugin

项目介绍

本次作业项目为一个maven插件，功能是打包maven项目在一个zip中，其中包含两项：

  1. jar包，为可执行jar包，可以通过以下方式进行运行：

    java -jar use-maven-project.jar

    jar包中是所有可执行程序的class文件及其需要的class文件

  2. lib目录，其中包含了jar包中所有要的依赖

过程描述

首先明确有哪些步骤：
(1) 创建项目
(2) 通过mavenProject获取所有的依赖jar包
(3) 创建zip包，即最终所需要的文件
(4) 创建jar包，即最终要执行的文件包
(5) 移动jar包到zip包
(6) 移动lib到zip包
(7) 在使用该插件的项目中进行注册，使用该插件

*接下来将介绍各个过程的具体步骤*

  1. 创建项目

    • packaging为maven-plugin
    • maven.compiler.source和maven.compiler.target统一设置成1.8。encoding设置成utf-8
    • 依赖加入如下三个
    • 构建工具引入如下工具

  2. 通过mavenProject获取所有的依赖jar包

    通过使用getDependencyArtifacts()获取所有的依赖文件并形成File对象集合，通过这些File对象获取名字后，通过字符串拼接形成Class-Path。这个过程不仅是获取依赖的jar包，同时也是为后面构建Manifest文件做准备。

    创建Manifest对象，里面包含了启动包时的函数入口，依赖位置，以及项目version等等等等。

    最后使用 manifest.write(outputStream)将这个对象写到jar包中（jarOutputStream）

  3. 创建zip包，即最终所需要的文件

    通过使用new File()直接创建zip文件，注意命名规范(这个在多人合作中非常重要哦！)，我采用的是直接与maven生成的jar包重名，但是后缀为zip。

  4. 创建jar包，即最终要执行的文件包

    在这一步中我们创建了一个方法：
    private void addToZipFile(File file, String entryName, ZipOutputStream zipOutputStream) throws IOException
    由于JarOutputStream继承自ZipOutputStream，所以用父类引用指向子类对象，所以就算是jarOutPutStream也能用这个函数。

    如何使用：将file放到zipOutPutStream中的entry文件夹下，所以我们现在可以通过这个函数自由的移动文件。

细节一览

  1. jar包是怎么构建起来的
    jar包中有一个文件叫META-INF/MANIFEST.MF文件
    见名知意，什么是manifest？就是展现。展现了jar文件打包过程中的一些数据，其中至少应该指出执行包的主函数位置，以及依赖项应当去哪里寻找。对应的key就是Main-Class和Class-Path，我们在2.中进行的拼接字符串的操作也就是为了创建Manifest做准备

  2. Main-Class找不到
    具体报错不是这个，有点记不起来了，大概就是说找不到你的主程序。有很多同学都遇到了这个问题。有的是斜杠打反了，有的balabala。
    我的是调用addToZipFile时没有完全弄懂这个函数中的功能，所以最后打包出来的jar中的内容为classes.com.ksyun.train.balabala,所以加上”com”就让其知道去哪里找，最后就是com.ksyun.train.balabala。但是就是这个问题困扰了我快一天的时间
    So! Be humble when you are coding!
    学习的时候谦虚点，不浮躁(再次有点小感悟)
    同时也有同学问了我这个问题，最后是系统的问题，他的是win，我的是mac，最后他用wsl跑了一下就好了，以后如果用win出现这个问题可以请教那个同学嘻嘻嘻。
