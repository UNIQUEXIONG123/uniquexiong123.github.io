本博客用于记录在实现利用反射的java参数解析器时的具体细节，以及总结过程以后遇到类似问题少走弯路

项目名称

param-parse-util

项目介绍

要求：

String queryString=”Cpu=2&Memory=4.0&AutoCreated=null&Container.1.Name=nginx&Container.1.Command.1=/bin/sh”

请根据规则将上述queryString字符串自动映射到对象的字段上，以下是参数接收对象，对象中的属性只提供了get方法，不提供set方法，请通过反射设置对应的属性值，Pod对象默认提供了无参构造方法。(这里就不展示pod对象，因为本程序要求能够处理所有自定义类型，容器只能处理ArrayList类型，可以嵌套)

  1. parse函数是本程序暴露的唯一接口，其的传递参数是一个class和一个String，不方便递归，所以写了一个新的函数parseParams，参数是class和Map<String[], String> params，params就是将queryString处理之后的结果

    map的具体形式如下:

  2. 遍历class类中的所有field，依次判定params中是否有传进来的参数，如果没有就跳过，如果有就进行后续的处理

为什么不遍历params，然后从class的getFields中的field进行匹配呢？
我主要考虑了这个方面：

  • 如果一次从params中取出一个变量，然后用所有的params去匹配，相当于要用一个数组去储存fields，但是fields不是一个简单的字符串，如果用数组，对比如果遍历params（map，里面是字符串），代价就要大得多
  1. 如何处理字符串？有三种情况：
    1. 基本数据类型和String以及Bigdecimal，这种情况首先判断具体是哪种类型，直接赋值就可以。这个过程用到了

    private static Object findType(Class<?> genericType, Map<String[], String> params, String[] key)

相当于把解析的字符串传进去，其返回一个确定的值，如果是”null”, “Null”, “NULL”,就赋值上默认值。

    1. [“Command”, “1”]这种类型，即基本数据类型的ArrayList形式
    1. [“Container”, “1”, “Name”]，这种形式表示嵌套的基础数据类型

    分这三种情况进行讨论

    (1) 如果是基本数据类型和补充的两种基本数据类型，则通过

    Object findType(Class<?> genericType, Map<String[], String> params, String[] key)

    函数，获取实际的数据后返回给fieldValue，上面已经比较具体的说过。

    (2)如果是嵌套的基本数据类型，那么递归调用parseParam，参数怎么办呢？将原来的param去掉第一个就行，例如：

    命名memberParam的意思是是成员(member)的参数map集

    class参数的获取方法是通过
    Class<?> genericType = getGenericType(field)
    函数进行获取，将这两个参数传到parseParam中进行递归就可以。
    这个函数可以帮助我们获取到范型的类型

    (3) 最难的莫过于List类似的这种类型，即[“Container”, “1”, “Commane”, “1”]类型，首先创建包装类CustomList，在内部创建一个比较器，可以通过优先级进行排序，比如进来两个实体分别是Container.1和Container.2，不管他们进来的顺序怎么样，在列表中的顺序一定是数字小的在前面，有了这个包装类之后，先用包装类储存生成的各个实体，然后依次取出来存放到fieldValue中即可。如何生成实体？

    • 1 通过
      Class<?> genericType = getGenericType(field)
      函数获得范型类型

    • 2 创建一个新的params,假如原来的params如下

      原来的param： {
      [“Container”, “1”, “Command”, “Name”]: “name”;
      }
      那么创建一个新的param如下，去掉key中的前两个String：
      memberParam: {
      [“Command”, “Name”]: “mysql”
      }

      这样就退化到了情况(2),继续用情况(2)的方法处理剩下的工作就可以

    • 3 调用parseParam生成实体，继续使用customList对各个实体进行排序，之后赋值到fieldValue中即可

如果是基本数据类型的List怎么办？

即[“Command”, “1”]: “name”

当我们遇到这种情况就不用进行递归，直接使用for循环依次创建实体后放到customList当中进行排序，之后就赋值给fieldValue。

作业要求补充：

  1. 加了注解的要跳过
    直接使用

    field.isAnnotationPresent(SkipMappingValueAnnotation.class);

    判断是否有注解就行，然后跳过
    使用的函数是

    private static boolean isSkipped(Field field)

  2. 如果Key是小写，那么直接忽略：

    比如Cointainer.1.command.1=3
    那么需要创建Container实体，但是不会去创建command，
    如果有以下两个query：
    Cointainer.1.command.1=3
    Cointainer.1.Command.2=3
    一个小写一个大写怎么办？
    Container.1.command将会有一个成员
    如果只有以下参数怎么办？
    Cointainer.1.command.1=3
    那么将会生成Container，但是command将会为空
    如何实现？

    这个功能写在了

    private static List<String[]> getKey(Map<String[], String> params, String fieldName)

    函数中。使用正则表示进行判定：keys[0].matches(“^[A-Z].*”)

  3. 如果是null, Null, NULL,就赋值默认值，这个功能写到了

    private static Object findType(Class<?> genericType, Map<String[], String> params, String[] key)
