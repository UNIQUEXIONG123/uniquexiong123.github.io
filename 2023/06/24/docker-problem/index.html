创建自己的Dockerfile

scratch image

scratch 可能是docker中最小的镜像。事实上，这个镜像是空的，里面不含有任何文件或者文件夹，它仅仅是构建我们镜像的一个起始点。

为了在scratch上面运行二进制文件，我们的可执行程序需要静态编译和自包含。意味着图像中没有编译器，只剩下系统调用。

你也不能够登陆进入容器，毫无疑问，其中也没有shell，除非你明确增加一个。

Docker0

  • docker0 是 Docker 在主机上创建的默认网络接口。当你在主机上安装并启动 Docker 时，它会自动创建一个名为 docker0 的虚拟网络接口。

  • docker0 接口是 Docker 使用的网桥（bridge）接口，它用于连接主机和运行在 Docker 容器中的应用程序之间的网络通信。它充当了主机和容器之间的网络桥梁。

  • docker0 网桥会自动配置一个 IP 地址，并且负责为容器分配 IP 地址，并与主机上的其他网络接口进行通信。

通过 ifconfig 命令可以查看 docker0 接口的详细信息，例如 IP 地址、子网掩码等：

使用容器名进行通信

docker run -d -P –name tomcat03 –link tomcat02 tomcat

使用”–link”可以通过容器名进行通信，但是只能正向通信，不能反向通信（因为没有配置）注册在了”/etc/hosts”中了。

由于不能通过容器名进行通信，因此一般不用docker0，很少使用”–link”

查看docker中所有网络

docker network ls
