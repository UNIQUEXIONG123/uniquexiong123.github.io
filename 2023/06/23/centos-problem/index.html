本博客记录一些学习linux过程中遇到的问题

Docker 环境问题

CentOS

报错：Failed to download metadata for repo appstream – CentOS 8
Error: Failed to download metadata for repo ‘appstream’: Cannot prepare internal mirrorlist: No URLs in mirrorlist

  1. 解释： 2021年12月31日，CentOS 8终结了，并且CentOS 8并不会自动迁移至下一个版本（CentOS Stream 8），因此需要手动进行迁移。
  2. 做法：迁移至新版本

Ubuntu

报错bash: sudo: command not found

按照如下顺序执行：

apt-get update
apt-get install sudo

  1. 没有vim装vim：

    sudo apt-get install vim

  2. ubuntu查看ip：

  • 如果要执行ip命令，安装iproute2软件包:

    apt install iproute2

  • 查看IP地址：

    ip a

  • 执行ifconfig命令,安装net-tools工具包:

    apt install net-tools

  • 查看IP地址：

    ifconfig

Docker：

  1. 安装docker

    curl -fsSL https://test.docker.com -o test-docker.sh
    sudo sh test-docker.sh

  2. 使用docker：
    如果出现类似于这样的报错

    System has not been booted with systemd as init system (PID 1). Can’t operate.

这意味着容器中的linux没有privilege，解决办法是在启动的时候就给予优先级，如下：

  • 创建新容器：

    docker run -it –privileged container-name /bin/bash

  • 启动已有容器：

    docker exec -it –privileged container-name /bin/bash
